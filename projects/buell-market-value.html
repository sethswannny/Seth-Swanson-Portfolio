Machine Learning Case Study

Buell Market Value Predictor

I wanted a public project that felt closer to a real pricing and deal-screening problem than a stock tutorial dataset. This one predicts motorcycle listing prices, then translates the prediction into a simple deal label a buyer or marketplace analyst could actually use.

Problem

Could a model spot whether a Buell listing is fairly priced?

The goal was not to make an official appraisal tool. The goal was to show the full data science workflow: define the decision, build a safe dataset, train multiple models, compare results, explain drivers, and turn the output into something usable.

Approach

Built the market first, then trained against it.

I generated listings for Buell models and comparable motorcycles using variables a marketplace would plausibly expose: mileage, condition, seller type, region, season, title status, performance specs, service records, listing photos, and description quality.

What I Would Improve

The honest limitations matter.

The current dataset is synthetic, so the model proves workflow more than market truth. If I expanded this, I would compare against public valuation guides, add real listing snapshots where licensing allows it, track time-based price drift, and separate Buell-only performance from broader motorcycle comparables.

Artifacts

Everything is reproducible.

The project includes a data generator, training pipeline, saved model, model metadata, charts, report, data dictionary, and Streamlit app. The code is structured so a reviewer can rerun the whole workflow.
