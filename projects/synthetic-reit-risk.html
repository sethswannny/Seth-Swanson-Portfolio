Synthetic REIT Portfolio Risk Model

A portfolio-safe classifier that screens fictional REIT assets into healthy, watchlist, and at-risk groups using lease, financial, tenant, and market signals.

Problem

Which assets need portfolio attention?

The model classifies assets into health buckets so a portfolio team can prioritize review. The point is not to create a production credit model; it is to show the workflow behind explainable asset health analytics without exposing private data.

Approach

Built a diversified synthetic portfolio.

The generated dataset includes industrial, grocery, quick service restaurant, medical office, automotive, convenience store, fitness, entertainment, office, and distribution assets across regions, credit tiers, rent coverage profiles, lease terms, delinquency, and market trends.

Modeling

The baseline won, which is useful.

Logistic regression beat the tree-based models on macro F1. That makes the project more credible: the simplest model was easier to explain and performed best on class balance.

What I Would Improve

Production would need governance.

A real version would require validated source systems, historical outcomes, time-based validation, threshold governance, portfolio manager review, and monitoring for drift. This project is deliberately a safe public workflow demo.

Artifacts

Everything is reproducible.

The project includes a synthetic data generator, training pipeline, saved classifier, confusion matrix, feature importance, watchlist output, Streamlit app, and standalone one-file report.
